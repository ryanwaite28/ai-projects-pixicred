**Thinking:**
Root cause: `profile = "rmw-llc"` in all three Terraform provider blocks forces the AWS SDK to look for a named credentials profile in `~/.aws/credentials`. In GitHub Actions, credentials are injected as environment variables via OIDC (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`) — there is no `~/.aws/credentials` file, so the profile lookup fails even though valid credentials exist in the environment. The fix: make `profile` a variable that defaults to `"rmw-llc"` locally and is overridden to empty string in CI. When the variable is empty, the provider block passes `null` for `profile`, which causes Terraform to use the default credential chain (environment variables → IAM role). The `dynamodb_table` deprecation warning is left as-is — it's non-blocking and switching to `use_lockfile` is a meaningful infrastructure change not requested.
